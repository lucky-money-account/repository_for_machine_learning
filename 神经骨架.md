## 激活函数

### 1. 常见激活函数的种类、特点及意义

| 激活函数       | 特点                                                         | 意义                                          |
| :------------- | :----------------------------------------------------------- | :-------------------------------------------- |
| **Sigmoid**    | 输出范围 (0,1)，平滑单调；易饱和导致梯度消失，非零中心（输出恒正） | 适合二分类输出层，将实数映射为概率            |
| **Tanh**       | 输出范围 (-1,1)，零中心；仍存在饱和区梯度消失                | 常用于循环神经网络或隐藏层，比Sigmoid梯度更强 |
| **ReLU**       | 计算简单，非饱和，加速收敛；负半轴输出0，可能造成神经元死亡  | 目前最常用的隐藏层激活函数，解决梯度消失      |
| **Leaky ReLU** | 负半轴有微小斜率，缓解Dying ReLU                             | 保留负值信息，提升模型表达能力                |
| **ELU**        | 负半轴指数衰减，输出均值接近0，更平滑                        | 结合ReLU优点，对噪声鲁棒，但计算稍复杂        |
| **Softmax**    | 输出和为1的概率分布，用于多分类输出层                        | 将logits转换为类别概率，便于交叉熵损失        |

### 2. 为什么需要非线性激活函数？如果没有会怎样？

- **原因**：线性变换的组合仍然是线性变换。若全部使用线性激活（或无激活），无论多少层神经网络都可等效为一个单层线性模型，无法逼近复杂的非线性函数（如 XOR 问题）。

- **作用**：非线性激活函数引入非线性能力，使神经网络能拟合任意复杂函数（万能近似定理）。

  ps.万能近似原理：一个**前馈神经网络**（例如多层感知机 MLP），如果至少包含一个**隐藏层**，并且隐藏层中使用**非线性激活函数**（如 Sigmoid、ReLU 等），那么只要隐藏层的神经元数量**足够多**，这个网络就可以以任意精度逼近**任何一个连续函数**。简单来说，只要你给足神经元，单隐层的神经网络就能模拟出任何复杂的曲线或曲面。但是效率可能非常低，因此我们引入了深度学习网络。

### 3. 什么是 Dying ReLU？如何解决？

- **定义**：ReLU 在负输入时梯度为0，若某些神经元持续接收负值，其参数将不再更新，永久死亡，输出始终为0。
- **解决方法**：
  - 使用 **Leaky ReLU** 或 **PReLU**（参数化ReLU），为负半轴赋予小梯度。
  - 使用 **ELU** 或 **SELU**。
  - 合理初始化（如He初始化）和调整学习率，避免大量负输入。

### 4. 梯度消失与梯度爆炸

- **梯度消失**：深层网络中，梯度通过链式法则反向传播时，若激活函数导数小于1（如Sigmoid），多次连乘后梯度指数衰减，导致底层参数几乎不更新。

- **梯度爆炸**：若权重初始化过大或网络过深，梯度连乘 >1，导致参数更新过大，训练不稳定。

- **比喻理解**：

  想象一个多级瀑布：

  - **梯度消失**：瀑布逐级下降，最后落到最下层的水流已经极其微弱，无法推动水车（参数更新）。
  - **梯度爆炸**：瀑布突然遇到陡崖，水流猛冲，冲毁水车。

- ps.梯度递推计算：从输出层向输入层逐层传递，每一层梯度会乘以该层激活函数的导数和权重

  ![QQ_1773752507538](../../AppData/Local/Temp/QQ_1773752507538.png)

- **常见解决方案**：

  - 使用ReLU族激活函数。
  - 合理初始化（Xavier/He初始化）。
  - 批归一化（Batch Normalization）。
    - 对每层输入进行标准化，使输出均值为 0、方差为 1，稳定梯度分布，减少对初始化的依赖，允许使用更大学习率。
    - 实验证明能有效缓解梯度消失/爆炸，并加速收敛。
  - 残差连接（ResNet）。
    - ResNet 引入“跳跃连接”让梯度直接回传，避免中间层梯度衰减。
  - 梯度裁剪（gradient clipping）。
    - 对于梯度爆炸，设定阈值，当梯度范数超过阈值时缩放到阈值范围内。

## 优化器与学习率

### 1. 常见优化器种类、特点及意义

| 优化器       | 特点                                        | 意义                                       |
| :----------- | :------------------------------------------ | :----------------------------------------- |
| **SGD**      | 基础梯度下降，可能震荡，收敛慢              | 简单，易于理解，适合小规模数据             |
| **Momentum** | 引入动量项，加速收敛，减少震荡              | 利用历史梯度方向，跳出局部极小             |
| **Adagrad**  | 自适应学习率，对稀疏特征效果好              | 为每个参数分配不同学习率，适合处理稀疏数据 |
| **RMSprop**  | 解决Adagrad学习率过快衰减，使用指数移动平均 | 适合非平稳目标，常用于RNN                  |
| **Adam**     | 结合Momentum和RMSprop，自适应学习率         | 广泛使用，收敛快，鲁棒性强                 |

### 2. 为什么要乘以学习率，而不是直接用梯度？

- 梯度指示了损失下降最快的方向，但步长（学习率）需控制。若步长过大，可能越过最优点导致震荡或发散；步长过小则收敛过慢。学习率作为缩放因子，平衡收敛速度与稳定性。

### 3. 学习率调度器的作用

- 动态调整学习率，常见策略：
  - **阶梯下降**：每固定epoch数衰减。每经过固定epoch数，学习率乘以一个衰减因子。
  - **指数衰减**：连续衰减。
  - **余弦退火**：按余弦函数周期变化。
  - **Warmup（预热）**：训练初期使用较小学习率，然后逐步增加到初始学习率，常用于Transformer等模型。
- **意义**：初期用较大学习率快速收敛，后期用小学习率精细搜索最优解，避免震荡，提升模型性能。

## 损失函数

### 1. 常见损失函数种类、特点及意义

| 损失函数                    | 特点                           | 意义                                 |                        |                            |
| :-------------------------- | :----------------------------- | :----------------------------------- | ---------------------- | -------------------------- |
| **均方误差（MSE）**         | 对大误差惩罚大，平滑可导       | 适用于回归问题，假设误差服从高斯分布 |                        |                            |
| **平均绝对误差（MAE）**     |                                |                                      | 对异常值鲁棒，梯度恒定 | 回归问题，尤其存在离群点时 |
| **交叉熵（Cross-Entropy）** | 衡量分布差异，梯度与误差成正比 | 分类问题，输出为概率时常用           |                        |                            |
| **Hinge Loss**              | 用于最大间隔分类               | SVM常用                              |                        |                            |
| **对数损失（Log Loss）**    | 逻辑回归损失                   | 二分类问题                           |                        |                            |

### 2. 为什么分类用交叉熵，回归用MSE？

- **分类问题**：输出通常是概率（经Softmax），交叉熵能有效衡量真实分布与预测分布的距离，且梯度形式简单（y^i−yi*y*^*i*−*y**i*），利于优化。若用MSE，梯度会乘以概率的导数，可能饱和，导致训练慢。
- **回归问题**：目标值是连续实数，MSE假设误差服从高斯分布，对应最大似然估计；且可导、凸性好，适合梯度下降。

## 泛化与正则化

### 1. 训练集、验证集、测试集的作用与划分

- **训练集**：用于学习模型参数。
- **验证集**：用于模型选择、超参数调优（如学习率、层数），评估模型在未见数据上的性能，防止过拟合。
- **测试集**：最终评估模型泛化能力，不参与任何训练或调参。
- **划分逻辑**：通常随机划分，如70%训练，15%验证，15%测试；或交叉验证。需保证数据分布一致。

### 2. 过拟合与欠拟合

- **过拟合**：模型在训练集上表现极好，但在测试集上表现差，即泛化能力弱。模型过度学习了训练数据中的噪声和细节。
- **欠拟合**：模型在训练集上表现差，未能捕捉数据规律，模型过于简单。

### 3. 常见解决方案及意义

| 问题   | 解决方案                 | 意义                         |
| :----- | :----------------------- | :--------------------------- |
| 过拟合 | 增加数据量               | 提供更多样本，减少过拟合风险 |
|        | 降低模型复杂度           | 减少参数，防止记忆噪声       |
|        | 正则化（L1/L2、Dropout） | 限制模型复杂度，提高泛化     |
|        | 早停                     | 验证集性能不再提升时停止训练 |
|        | 数据增强                 | 增加训练样本多样性           |
| 欠拟合 | 增加模型复杂度           | 提升表达能力                 |
|        | 减少正则化强度           | 允许模型更灵活               |
|        | 训练更长时间             | 充分学习                     |
|        | 改进特征或架构           | 更合适的数据表示或模型       |

### 4. 什么是正则化？常见手段

- **正则化**：在损失函数中加入惩罚项，限制模型复杂度，防止过拟合。
- **常见手段**：
  - **L1正则化**：λ∑∣w∣，产生稀疏权重。
  - **L2正则化**：λ∑（w^2），使权重平滑（权重衰减）。
  - **数据增强**：对原始数据做变换（旋转、翻转等），增加样本多样性。
  - **Dropout**：训练时随机丢弃部分神经元，防止依赖特定特征。
  - **Batch Normalization**：规范化每层输入，加速训练且有轻微正则化效果。

### 5. 为什么L1/L2正则化能减缓过拟合？

- L1/L2通过惩罚大权重，迫使模型学习更简单的模式，避免过于复杂。L1可产生稀疏解，移除不相关特征；L2使权重均匀减小，降低模型对个别特征的敏感度，从而提升泛化能力。

### 6. （挑战）从几何角度解释L1产生稀疏权重，L2使参数平滑

- 考虑二维权重空间 w1,w2*w*1,*w*2，损失函数的等高线是椭圆形，正则化项构成约束区域：
  - **L1**：约束区域是菱形（∣w1∣+∣w2∣≤C），其顶点在坐标轴上。最优解容易落在顶点，导致某些权重为0，即稀疏。
  - **L2**：约束区域是圆形（w1^2+w2^2≤C），最优解在圆周上，通常两个权重都非零且成比例缩小，即平滑。

## 数据预处理

### 1. 数据标准化与归一化

- **标准化（Standardization）**：将数据转换为均值为0，标准差为1
- **归一化（Normalization）**：将数据缩放到特定区间，如[0,1]

### 2. 不标准化对梯度下降的影响（等高线图）

- 若特征尺度差异大，损失函数的等高线会呈狭长椭圆形。梯度下降时，路径会沿陡峭方向震荡，缓慢向最优点移动。标准化后，等高线接近圆形，梯度直接指向圆心，收敛更快且稳定。

  椭圆越狭长，意味着沿长轴方向梯度很小（梯度垂直于等高线，在长轴方向梯度分量弱），沿短轴方向梯度很大。梯度方向垂直于等高线，指向最陡下降方向。在狭长椭圆情况下：

  - 在椭圆陡峭的短轴方向，梯度很大，参数更新步长大，容易越过最优值，导致来回震荡。
  - 在平缓的长轴方向，梯度很小，参数更新缓慢，需要多次迭代才能沿长轴移动。

  因此，梯度下降的轨迹会呈现“之”字形震荡，沿着椭圆长轴缓慢爬行。收敛速度极慢，且需要精心调小学习率来避免发散，这进一步拖慢进程。

## 评价指标

### 1. 常见评价指标及意义

| 指标                    | 公式                                     | 意义                                     |
| :---------------------- | :--------------------------------------- | :--------------------------------------- |
| **Accuracy（准确率）**  | (TP+TN)/(TP+TN+FP+FN)                    | 所有样本中正确分类的比例                 |
| **Precision（精确率）** | TP/(TP+FP)                               | 预测为正例中实际为正的比例，衡量查准率   |
| **Recall（召回率）**    | TP/(TP+FN)                               | 实际为正例中被正确预测的比例，衡量查全率 |
| **F1-score**            | 2*(PRECISION *RECALL)/(PRECISION+RECALL) | Precision和Recall的调和平均，综合两者    |

### 2. 为什么样本不均衡时Accuracy会“骗人”？

- 例如99%负例，1%正例，模型若全部预测为负，Accuracy高达99%，但无法识别任何正例，失去实际意义。因此需结合Precision、Recall等指标评估少数类性能。



# Conclusion：

神经网络的训练过程本质上就是**通过梯度下降来最小化损失函数**，从而找到一组最优的参数（权重和偏置）。这个过程可以拆解为几个关键环节：

1. **目标**：让损失函数 L(θ)*L*(*θ*) 的值尽可能小，其中 θ*θ* 代表所有参数。损失函数衡量了模型预测与真实标签之间的差距。

2. **核心工具**：**梯度** ∇L(θ)∇*L*(*θ*) 告诉你损失函数在当前参数下上升最快的方向，因此反方向就是下降最快的方向。（局部贪心）

3. **优化器**：它决定了**如何利用梯度来更新参数**（比如 SGD、Adam 等），包括步长（学习率）、动量等，影响着下降的效率和稳定性。

4. **学习率调度器**：它动态调整步长，帮助模型在初期快速下降，在后期精细搜索，避免震荡或陷入局部极小。

5. **激活函数**：引入非线性，使网络能够拟合复杂的函数，否则多层线性叠加仍等价于单层线性模型，无法解决复杂问题。

6. **反向传播**：高效地计算梯度，通过链式法则将损失对输出的误差层层传回，从而得到每个参数的偏导数。

7. **最终的目的是让模型在未知数据上也能表现良好，即具有好的泛化能力。**（可推广性）

   

   

# 深度学习训练流程：

1. 取一个批次的数据。
2. 前向传播得到预测。
3. 计算损失。
4. 反向传播计算梯度。（直接调用函数：`loss.backward()`）
5. 根据梯度更新参数。（直接调用函数：`optimizer.step()`）
6. 清空梯度，准备下一个批次。（直接调用函数：`optimizer.zero_grad()`）
7. 重复直到所有批次都处理完（一个 epoch 结束）。
8. 进行下一个 epoch，重复上述过程。



# 深度学习模型调用：

## 1. 切换模型到评估模式（`model.eval()`）

训练结束后，记得调用 `model.eval()`。这会通知模型进入**评估模式**，主要影响两个地方：

- **Dropout**：在评估模式下，Dropout 会被关闭，所有神经元都参与计算，不再随机丢弃。
- **Batch Normalization**：会使用训练阶段统计好的均值和方差，而不是根据当前批次重新计算。

这一步很重要，否则测试结果可能不稳定或不准。

------

## 2. 禁用梯度计算（`torch.no_grad()`）

在测试时，我们不需要计算梯度（因为不进行参数更新），所以可以用 `torch.no_grad()` 上下文管理器包裹预测代码，这样可以节省显存和计算时间，同时避免意外梯度累加。

python

```
with torch.no_grad():
    # 在这里进行所有测试相关的计算
```



------

## 3. 对测试集进行预测

假设你已经有了测试数据 `test_loader`（一个 `DataLoader`），里面包含了测试样本 `test_x` 和对应的标签 `test_y`。

你需要：

- 遍历所有测试批次。
- 对每个批次，将数据传入模型，得到输出 `outputs`。
- 通常模型的输出是每个类别的得分（logits），用 `torch.argmax(outputs, dim=1)` 得到预测的类别索引（0~9）。
- 将预测结果与真实标签比较，统计正确个数，最后计算准确率。

------

## 4. 完整代码示例

以下是一个完整的测试代码片段，紧接在训练循环之后：

python

```python
# 训练循环结束后
model.eval()   # 切换到评估模式

correct = 0
total = 0

with torch.no_grad():   # 测试时不需要梯度
    for batch_x, batch_y in test_loader:
        # 前向传播
        outputs = model(batch_x)
        
        # 预测类别（取最高得分的索引），返回预测结果
        _, predicted = torch.max(outputs, 1)   # 或 torch.argmax(outputs, dim=1)
        #predict代表的就是对于该样本的预测label值，我们不需要原始数据中的可能性大小的数据。
        # 统计正确数
        total += batch_y.size(0)	#累加样本总量
        correct += (predicted == batch_y).sum().item()	#正误比较并累加
        #.sum().item() 的作用
		.sum() 对布尔张量求和，True 被当作 1，False 为 0，得到该批次中正确预测的数量（一个整数张量）。.item() 将这个整数张量转换为 Python 整数（例如 2），方便累加到 correct 变量中。

# 计算准确率
accuracy = 100.0 * correct / total
print(f'测试集准确率: {accuracy:.2f}%')
```



**说明：**

- `torch.max(outputs, 1)` 返回每行最大值和对应的索引，我们只需要索引 `predicted`。
- `(predicted == batch_y).sum()` 得到一个批次中预测正确的数量，`.item()` 将其转为 Python 数值。
- 累加所有批次后计算总体准确率。

------

## 5. 注意事项

- **数据格式**：确保传入模型的数据类型和维度与训练时一致。通常需要是 `torch.float32`，形状为 `(batch_size, channels, height, width)`（对于图像）或 `(batch_size, 784)`（对于展平的图像）。
- **设备**：如果你的模型在 GPU 上训练，测试数据也要放到同一设备上（使用 `.to(device)`）。
- **标签**：标签通常是整数（0~9），可以直接与预测比较。
